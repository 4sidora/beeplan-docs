# BeePlan — требования (черновик)

## Цель продукта

Пчеловод видит телеметрию семей (температура, влажность, акустические признаки и далее вес и др.), ведёт пасечный журнал и в перспективе получает системные события на основе анализа данных.

## Компоненты

1. **EdgeDevice** — ESP32 в улье, датчики (MAX9814, температура/влажность, позже вес), глубокий сон между замерами, передача на концентратор по **ESP‑NOW**.
2. **Concentrator (Gateway)** — ESP32 с GSM (например T‑Call) или Wi‑Fi для разработки; приём ESP‑NOW, буфер на flash, периодическая отправка **HTTPS JSON** на сервер.
3. **beeplan-api** — REST API, версия пути `/v1`, хранение в PostgreSQL.
4. **beeplan-web** — SPA для просмотра данных и управления сущностями.

## Доменная модель (целевая, с заделом)

- **User** — владелец данных; позже OAuth Google/Яндекс.
- **Apiary** — пасека; принадлежит пользователю.
- **Concentrator** — привязан к пасеке; имеет секрет для приёма телеметрии от поля.
- **Colony** — пчелиная семья; принадлежит пасеке; может перемещаться между пасеками (позже). Поля MVP в веб/API: `name` (автогенерация «Имя 034» при создании, редактируемое), `description`, `bee_breed` (справочник), `colony_type` (отводок / нуклеус / семья / рой), `hive_type` с `body_count` и `frames_per_body` или `hive_volume_m3` для колоды.
- **EdgeDevice** — привязан к концентратору. В каждый момент времени **не более одной** активной семьи: `current_colony_id` (nullable). Телеметрия относится к семье на момент замера.
- **История назначения устройства семьям** — таблица интервалов (`device_id`, `colony_id`, `attached_at`, `detached_at`); не связь many‑to‑many «все сразу», а аудит переносов.
- **TelemetrySample** — `colony_id`, `metric`, `ts`, значение (JSON или скаляры); опционально `source_device_id`.

Типы ульев (справочник в коде): лежак Дадан (16/20/24 рам.), Дадан, Рута (10), магазинка, удав (9), МФУ (8), колода (объём м³). Журнал пасеки, системные события (роение, потеря матки и т.д.), ML по акустике — **после MVP**, поля и таблицы закладываются по мере этапов.

## Телеметрия MVP

- Метрики: `temperature_c`, `relative_humidity`, `audio_features` (компактный вектор признаков с edge).
- Частоты опроса настраиваемые (например 10 мин на улей, 1 ч uplink с концентратора).

## Нефункциональные требования

- Open source (лицензия MIT или Apache‑2.0 — выбрать при первом релезе).
- API контракт в **OpenAPI** в репозитории `beeplan-api`.
- Расширяемый транспорт улей ↔ концентратор (сейчас ESP‑NOW).

## Безопасность

- Секреты концентратора и JWT не коммитить; использовать `.env` и переменные окружения.
- См. [SECURITY.md](https://github.com/4sidora/beeplan-api/blob/main/SECURITY.md) в репозитории **beeplan-api**.
