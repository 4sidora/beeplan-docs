# BeePlan — инструкции для агентов и разработчиков

## Общие правила

- Название продукта и публичный бренд: **BeePlan**. В коде Python пакет `beeplan`, репозитории `beeplan-*`.
- Не расширяйте область задачи без явного запроса: сначала минимально рабочий путь, затем рефакторинг.
- **Не коммитьте** `.env`, ключи API, пароли, PEM, `local_settings` с секретами.
- Обратная совместимость API: путь версии `/v1/`; ломающие изменения — новая версия (`/v2/`) или согласованный deprecation.

## Контракты

- Источник правды по HTTP: `beeplan-api/docs/openapi.yaml` в репозитории **beeplan-api**. Изменения эндпоинтов сопровождайте обновлением OpenAPI и краткой записью в [ARCHITECTURE.md](ARCHITECTURE.md) в этом репозитории при смене потоков данных.
- Прошивки: магическая константа и версия бинарного протокола ESP‑NOW документируются в README репозиториев **beeplan-edge** и **beeplan-gateway** одинаково.

## Edge / Gateway

- Изменения схемы железа, пинов, шагов настройки связи или поведения радио/API — **в том же изменении** обновляйте [HARDWARE.md](HARDWARE.md) в этом репозитории.
- Разработка без железа: выносите CRC, упаковку пакетов и преобразование в JSON в функции с unit‑тестами на host (где возможно) или тесты на QEMU не обязательны для MVP.
- Энергосбережение на edge: избегайте busy‑loop; предпочитайте `esp_light_sleep`/`esp_deep_sleep` по плану замеров.

## Backend

- Стек: FastAPI, SQLAlchemy 2, Alembic, PostgreSQL.
- Миграции: только через Alembic; не правьте прод‑схему руками без миграции.
- Ingest телеметрии должен быть **идемпотентным** на уровне бизнес‑правил там, где возможны повторы сети (позже: `client_batch_id`); в MVP допускается дублирование записей — помечайте TODO в коде.

## Web

- TypeScript strict по возможности; общие типы можно генерировать из OpenAPI позже (`openapi-typescript`).
- Тексты UI: подготовка к i18n (ключи строк), язык по умолчанию русский.

## Тесты и CI

- API: хотя бы smoke‑тесты на `GET /health` и создание замера через batch с тестовой БД.
- PR не должен ломать `docker compose build` для `beeplan-api`.
